# MVP 开发时间规划（8 周）

## 总体节奏

| 阶段 | 时间 | 主题 |
|------|------|------|
| 阶段 0 | Week 1 | 环境搭建 + 接口对齐 |
| 阶段 1 | Week 2-3 | P0 核心开发（MVP 骨架） |

## Week 1：环境搭建 & 接口对齐

**全周目标**：所有人开发环境就绪，接口规范确认完毕。

| 成员 | 任务 | 交付物 |
|------|------|--------|
| A | 搭建 FastAPI 项目骨架；定义 Tool 接口规范（`agent/tools/base.py`）；与全员对齐 API 格式 | `backend/main.py` 可启动；`base.py` 完成 |
| B | 安装 PostgreSQL + TimescaleDB；编写 `init.sql`；与 A 确认 data_tool/alarm_tool 签名 | `init.sql` 可执行建表 |
| C | 初始化 Vue3 + Element Plus 项目；与 A 确认 API 响应格式；搭建 mock 数据层 | `frontend/` 工程可启动 |
| D | 安装 FAISS 等 RAG 依赖；与 A 确认 rag_tool 签名；开始收集知识文档素材 | RAG 环境就绪 |

**本周检查点（周五）**：
- [ ] 所有人能本地运行自己的模块
- [ ] A 的 `agent/tools/base.py` 已被 B、D 确认
- [ ] A 的 API 规范已被 C 确认
- [ ] GitHub develop 分支创建完成

## 风险预案

| 风险 | 影响 | 预案 |
|------|------|------|
| DeepSeek API 不稳定 | Agent 不可用 | 降级为本地 ChatGLM 或预定义规则 |
| Prophet 安装困难 | 预测功能缺失 | 降级为简单线性回归 |
| FAISS 检索效果差 | RAG 不可用 | 降级为关键词匹配 |
| 前端页面做不完 | 展示效果差 | 聚焦 AI 对话 + 监控面板，其余页面后续补 |
| 成员进度落后 | 联调延迟 | A（队长）协调补位 |
