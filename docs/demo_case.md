
# Demo Cases


## 场景1：异常诊断

用户输入：

> 分析2号机组主蒸汽温度异常

系统输出：

- 设备：2号机组
- 异常：主蒸汽温度持续升高
- 风险评分：0.87
- 可能原因：
  1. 减温水流量不足
  2. 调节阀异常
- 建议：检查减温水系统


---


## 场景2：趋势预测

用户输入：

> 预测2号机组未来6小时负荷趋势

系统输出：

- 设备：2号机组
- 预测时段：未来6小时
- 趋势：负荷预计从 320MW 逐步上升至 380MW
- 置信度：0.92


---


## 场景3：历史对比分析

用户输入：

> 对比昨天和今天14-16点锅炉主蒸汽温度

系统输出：

- 对比窗口：昨天14:00-16:00 vs 今天14:00-16:00
- 平均温差：+5.2°C
- 最大温差：+10.1°C
- 变化率：+0.87°C/h
- 结论：温度呈上升趋势，建议关注


---


## 场景4：故障知识检索

用户输入：

> 锅炉主蒸汽温度过高的常见原因有哪些？

系统输出（基于 RAG 检索）：

- 相关案例：3条
- 常见原因：
  1. 减温水调节阀故障（案例 #12，相似度 0.94）
  2. 燃料量异常增加（案例 #07，相似度 0.89）
  3. 过热器积灰（案例 #23，相似度 0.81）
- 参考规程：《锅炉运行规程》第3.2节


---


## 场景5：全流程智能诊断（端到端演示）

用户输入：

> 全面诊断2号机组当前运行状态

Agent 执行流程：

1. 参数解析 → 识别设备：2号机组，任务：全量诊断
2. 数据查询 → 拉取过去24小时所有测点时序数据
3. 异常检测 → 多策略检测（阈值/趋势/波动/残差）
4. 知识检索 → RAG 匹配历史故障案例和运行规程
5. 综合推理 → LLM 融合数据 + 知识，生成诊断结论
6. 报告生成 → 输出标准化诊断报告 + ECharts 趋势图

预计耗时：< 5 秒

输出示例：

====================================

        2号机组综合诊断报告
        
        生成时间：2026-07-11 14:30:00
        
====================================

【风险总览】
  综合风险评分：0.35（正常范围）

【异常项】
  1. 主蒸汽温度 — 轻微上升趋势 — 风险 0.62
  2. 轴承振动 — 正常

【建议措施】
  1. 持续监测主蒸汽温度变化
  2. 检查减温水系统阀门开度
  3. 如温度继续上升，建议降低负荷

【关联案例】
  历史相似案例 #12：主蒸汽温度异常（相似度 0.87）

====================================
