  本系统是一个基于 Multi-Agent 的电厂智能预警与故障诊断系统，包含数据层、算法层、Agent层、接口层四个部分。


  ## 总体架构图：

  ```mermaid
  flowchart LR

  %% ========== 用户层 ==========
  U[用户 / 运维人员]
  F[前端系统 Frontend<br/>Vue/React]

  %% ========== 接口层 ==========
  API[API网关 / 后端服务<br/>FastAPI]

  %% ========== Agent层 ==========
  A1[调度Agent]
  A2[诊断Agent]
  A3[报告生成Agent]

  %% ========== 算法层 =======1===
  M1[异常检测模型<br/>LSTM/阈值]
  M2[故障分类模型<br/>MLP/规则]

  %% ========== 数据层 ==========
  DB1[(时序数据库<br/>InfluxDB)]
  DB2[(业务数据库<br/>MySQL)]

  %% ========== 数据源 ==========
  DS[传感器/历史数据]

  %% ========== 连接关系 ==========
  U --> F
  F --> API

  API --> A1
  A1 --> A2
  A2 --> A3

  A2 --> M1
  A2 --> M2

  M1 --> DB1
  M2 --> DB1

  API --> DB2

  DS --> DB1
  ```
  

  ## 流程：
  指令输入 → 智能解析 → 自主执行（数据查询、预警研判、故障诊断等） → 结果反馈 → 模型优化




  ## 模块划分：
  ### 1.指令解析与交互模块
  
      A负责部分：
      
        1.LLM Agent指令理解核心
        
          - DeepSeek API调用
          - ChatGLM/LLaMA本地部署尝试
          - Prompt设计
          
        2.Intent识别模块
        
        3.参数抽取模块
        
        4.Agent流程设计
        
      B负责部分：工业领域语义支持（准确提取机组、参数、时间、阈值）
      
        1.建立工业参数字典
        
        2.设备知识映射
        
          用户：
          
          2号机转换：
          
          {
          
            device_id:"generator_002"
            
          }

        3.提供解析测试数据
        
          制作100~200条工业指令，例如：查询2号机组温度，分析昨天压力异常，预测未来6小时负荷等
          
      C负责部分：交互界面与快捷指令
          对应需求：支持快捷键指令配置
        
        1. 快捷指令UI
        
          设计按钮[实时监控][异常分析][趋势预测][故障诊断][生成报告]等，点击自动生成（分析2号机组当前状态等），发送给Agent。
          
        2. 输入交互
        
          实现文本框，语音按钮，快捷菜单
          
        3. 展示解析结果
        
          用户输入“分析2号机组温度”，页面显示识别结果：
          
            设备：
            2号机组
            
            参数：
            主蒸汽温度
            
            时间：
            24小时
            
            任务：
            异常检测
          等内容
          
      D负责部分：
      
        1. 语音识别测试
        
          直接调用Whisper API，FunASR，负责整理测试集，验证准确率
          
        2. 解析效果测试
        
          测试输入，检查输出，统计准确的结果


    
 ### 2. 数据处理与分析模块
 
     主负责人：B
     
     A：系统架构与接口负责人
     
       1.数据查询接口封装
       
         B提供:
         
             query_timeseries()
             
         A封装成Agent Tool：
         
             data_query_tool()
             
       2：后端API设计
       
         提供给C接口
         
     B：工业数据负责人
     
       1：电厂运行数据管理（支持实时、历史数据调取与处理）
       
         数据库设计（PostgreSQL）
         
           设备表
           
           字段
           
           测点表
           
           TimescaleDB：
           
             时序数据表
             
             字段
             
       2：实时数据处理（目标：数据处理延迟 <3秒）
       
         模拟数据采集——>数据清洗——>入库——>查询
         
        处理：
        
          缺失值处理
          异常值过滤
          时间对齐
          
       3：趋势分析算法（自动生成参数趋势曲线）
       
         提供数据给C画图
         
       4：数据钻取逻辑
       
         设计三级查询
           第一层：整体趋势
           第二层：异常时间段
           第三层：分钟级数据
           
       5：任意时段数据对比
       
         例如：
          用户：对比昨天和今天14-16点温度
          处理：
          时间窗口A VS 时间窗口B
          计算：平均值，最大值，最小值，变化率
          输出：
          {
          "avg_diff":5.2,
          "max_diff":10
          }
          
      C：前端与产品负责人
      
        1：趋势曲线页面
        
          技术：Vue3 + Echarts
          支持：缩放，时间选择，参数选择
          
        2：数据钻取交互
        
        3：数据对比展示
        
      D：知识库与测试负责人
      
        1：数据测试
        
        2：分析结果验证
           
 ### 3. 预警与故障诊断模块
     A负责：Agent诊断流程集成
        1：预警Tool封装
          B提供：
            warning_service()
          A封装：
            warning_tool()
          Agent调用
          用户：分析2号机组当前异常
          Agent调用返回
        2：诊断流程编排
          实现：
          发现异常，调用预警Tool，调用RAG Tool，生成诊断报告，返回用户
        3：结果融合
      B负责：预警识别与趋势研判

  ## 数据流：
  用户请求 → API → Agent → 调用算法 → 查询数据库 → 返回结果


  ## 技术选型：
  后端：FastAPI
  模型：PyTorch
  数据库：MySQL / InfluxDB
  Agent：基于 LLM（如 DeepSeek / OpenAI）
